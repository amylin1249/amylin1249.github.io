Project All Access Livable Housing

Developed a spatio-temporal data application to model housing and homelessness trends in San Francisco.

Tools: Python (Pandas, Plotly Dash, Altair)
Course: Computer Science with Applications II
Links: Live Application, GitHub Repository

Overview#

Working in a team of four, we built an end-to-end data application to analyze housing and homelessness trends in San Francisco. I led the development of the core data engineering infrastructure, including a Python pipeline designed to ingest and standardize disparate public datasets. This cleaned, merged data fed into a dashboard and visualization suite that I designed and developed. By highlighting critical spatial and temporal patterns, this tool enables policymakers and community organizations to better target interventions, allocate resources, and make informed, data-driven housing policy decisions.

My Contributions#

Data Engineering Pipeline#

  • Robust Ingestion: Built a cleaning pipeline for three of the four primary datasets: median rent, encampment counts, and 311 service calls.
  • Temporal Interpolation: Harmonized inconsistent reporting cycles using linear interpolation to create a continuous, high-frequency time-series for trend analysis.
  • Spatial Crosswalking: Engineered a ZIP-to-census-tract mapping pipeline, utilizing weighted population aggregations to ensure granular data alignment across disparate geographic levels.
  • Statistical Imputation: Addressed missing data by implementing mean imputation across feature sets, maintaining sample size integrity and ensuring model robustness.

Data Visualization & UX#

  • Interactive Graphs: Developed line graphs visualizing how monthly median rent, encampment distributions, and street homeless population estimates change over time across specific zip codes and census tracts.
  • UX & Design Leadership: Led the dashboard interface design and authored textual content to ensure technical findings were accessible to policy stakeholders.
